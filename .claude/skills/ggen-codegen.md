# Skill: ggen Code Generation Pipeline

## Purpose

Understand how ggen transforms RDF ontologies into generated Rust code via SPARQL CONSTRUCT and Tera templates.

## Pipeline Stages (μ₁–μ₅)

```
.specify/*.ttl (source ontology)
  ↓ μ₁ (Load): Parse TTL, validate syntax
.specify/queries/*.rq (SPARQL CONSTRUCT)
  ↓ μ₂ (Extract): Execute CONSTRUCT, produce intermediate facts
intermediate-facts.rdf
  ↓ μ₃ (Generate): Tera templates consume facts, produce code
*.rs (generated code)
  ↓ μ₄ (Validate): SHACL shapes validate intermediate, type-check generated code
cargo check ✓
  ↓ μ₅ (Emit): Write files to src/cmds/generated.rs, cell8-ggen/src/cell8/generated/
```

## ggen.toml Configuration

```toml
[package]
name = "open-ontologies"
version = "0.1.0"

[ggen]
# Source ontology
ontology = "ontology/cli-open-ontologies.ttl"

# SPARQL query directory
queries_dir = ".specify/queries/"

# Template directory
templates_dir = ".specify/templates/"

# Output files
output_files = [
  { src = "cli.tera", dest = "src/cmds/generated.rs", format = "rust" },
  { src = "tools.tera", dest = "src/tools/generated.rs", format = "rust" },
]

# Optional: SHACL shapes for validation
shapes = "ontology/cell8-shapes.ttl"

# Profile for code generation
profile = "default"
```

## Example: Extract and Generate CLI Command

### 1. Source TTL (ontology/cli-open-ontologies.ttl)

```turtle
@prefix onto: <https://ggen.io/onto/cli/open-ontologies/> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

onto:ValidateCommand
  a onto:OntologyCommand ;
  rdfs:label "Validate" ;
  rdfs:comment "Validate ontology against SHACL shapes" ;
  onto:hasVerb onto:validate ;
  onto:hasOption onto:optShapes ;
  onto:hasOption onto:optFormat .

onto:optShapes
  a onto:Option ;
  rdfs:label "shapes" ;
  rdfs:comment "Path to SHACL shapes file" ;
  onto:isRequired true .
```

### 2. SPARQL Query (.specify/queries/extract-commands.rq)

```sparql
PREFIX onto: <https://ggen.io/onto/cli/open-ontologies/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

CONSTRUCT {
  ?cmd rdfs:label ?cmdName ;
       rdfs:comment ?cmdDesc ;
       onto:hasVerb ?verb ;
       onto:hasOption ?opt .
  ?opt rdfs:label ?optName ;
       rdfs:comment ?optDesc ;
       onto:isRequired ?required .
}
WHERE {
  ?cmd a onto:OntologyCommand ;
       rdfs:label ?cmdName ;
       rdfs:comment ?cmdDesc ;
       onto:hasVerb ?verb .
  OPTIONAL {
    ?cmd onto:hasOption ?opt .
    ?opt rdfs:label ?optName ;
         rdfs:comment ?optDesc ;
         onto:isRequired ?required .
  }
}
```

Produces intermediate facts:
```
onto:ValidateCommand rdfs:label "Validate" ;
                     rdfs:comment "Validate ontology..." ;
                     onto:hasVerb onto:validate ;
                     onto:hasOption onto:optShapes .
onto:optShapes rdfs:label "shapes" ;
               rdfs:comment "Path to SHACL shapes file" ;
               onto:isRequired true .
```

## Key Rules

### ❌ FORBIDDEN

- Manually editing `src/cmds/generated.rs` (it's an artifact)
- Running `ggen sync` without validating source TTL first (`onto validate`)
- Modifying generated code without updating source TTL
- Breaking chain: change TTL but skip `ggen sync`

### ✅ REQUIRED

- Edit source TTL in `ontology/`
- Run `onto validate` to verify syntax
- Run `ggen sync --dry-run true` to preview
- Run `ggen sync --audit true` for full execution with receipt
- Verify generated code compiles: `cargo make check`

## Workflow

```bash
# Step 1: Edit source ontology
vim ontology/cli-open-ontologies.ttl

# Step 2: Validate syntax and SHACL
onto validate ontology/cli-open-ontologies.ttl

# Step 3: Preview code generation
ggen sync --dry-run true

# Step 4: Execute full pipeline
ggen sync --audit true

# Step 5: Verify compilation
cargo make check

# Step 6: Commit
git add ontology/cli-open-ontologies.ttl src/cmds/generated.rs
git commit -m "feat(ontology): Update CLI commands via ggen"
```

## Commands

```bash
# Validate ontology before sync
onto validate ontology/cli-open-ontologies.ttl

# Preview code generation (no writes)
ggen sync --dry-run true

# Full pipeline with receipt
ggen sync --audit true

# Check generated code
cargo make check

# Clean regeneration
rm src/cmds/generated.rs && ggen sync --audit true
```
